# AP-30. Plausibility / structural bias (Layer 2 cross-doc bounded)

**Что нельзя:**

- Принимать AI artifact (ADR / plan / spec section) потому что он выглядит как «mature engineering» / «elegant generalization» / «architecturally complete», без cross-check на ground truth.
- AI generates artifact по pattern: «if есть wrap_1, wrap_2 — для symmetry должна быть wrap_3», «if есть layer_A, layer_B — добавим layer_C для completeness», «if есть N-1 tier — N-tier выглядит maturer».
- Visual signal of architectural quality (symmetric / layered / completed) ↑ overrides skepticism у PM-оператора.

**Почему:** Inverse correlation observed на live test'е 2026-05-25: visual signal quality ↑ → actual quality ↓ когда дополнительный tier hallucinated. «Three-tier envelope» звучит maturer чем «two-tier». Symmetric layered designs выглядят architecturally completer. PM подсознательно accept'ит «more architectural» version. AI generates по pattern, не по requirements ground truth.

**Failure mode common pattern:** AI optimizes for **looking** like mature engineering, не для **being** correctly-scoped engineering. Pattern bias к symmetric layered architectures preserves в training data — AI reproduces. PM не имеет cognitive bandwidth для verify каждое «elegant» design против spec / foundational invariants.

**Mode:** discretionary review check, не auto-detect. Surface as `[question]` finding, не blocking без operator-confirm.

**Как поступать вместо:**

1. **Reviewer Step 2.5 § 3.6.4 discretionary check** — spot patterns:
   - Symmetric layered designs где tier_3 / wrap_3 / level_3 не traced к concrete invariant.
   - «Mature engineering» framing без supporting source citation.
   - N+1 architectural extension «for completeness» без spec demand.

2. **При spot** → `[question]` finding tagged `AP-30`: «Component X выглядит как plausibility-driven extension (visual signal mature, но не traced к source). Verify with operator: invariant / scenario demanding это?»

3. **Conservative threshold:** sometimes catches legitimate engineering judgement. Surface as question, не blocking, до operator-confirm. Better surface ambiguous case чем silently accept.

4. **Bias acknowledgement в planner.md** — explicit reminder: «не add'ить component для symmetric completeness; trace к concrete spec / invariant required».

**No override needed** — finding is `[question]`, не blocking. Operator decides.

**Use case examples:**

- *Plausibility trap (what gets flagged):* ADR Decision вводит `wrap_session` потому что есть `wrap_master`, `wrap_device` — symmetric three-tier design. Reviewer flag question: «wrap_session не traced к scenario / invariant. Это plausibility-driven? Verify.»
- *Legitimate symmetry:* `wrap_session` traced к `<topic>_spec.md § Scenario: session loss recovery`. Source citation valid → no flag (even если visually symmetric).
